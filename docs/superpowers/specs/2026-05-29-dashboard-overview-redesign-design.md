# 首页概览化重构设计

## 目标

把首页从“所有内容堆在一起”改成“概览优先”的仪表盘结构：默认页只显示总览和可视化内容，完整列表和细节进入左侧导航对应的子视图。

## 范围

本轮只改前端信息架构：

- 首页默认打开“概览”。
- 左侧增加导航，切换 `概览`、`今日情报`、`观察雷达`、`来源状态`。
- 概览页只放总指标、运行状态、LLM 摘要和重点提醒。
- 今日情报页放筛选条、主线、完整条目列表。
- 观察雷达页放观察对象和历史走势。
- 来源状态页放来源健康、近期趋势和本周沉淀。

本轮不改抓取逻辑、不改数据库结构、不改 `/api/*` 语义。

## 设计原则

- 复用现有数据和 API，尽量只做页面重排。
- 先隐藏再重组，避免一次性改写整个页面。
- 默认页必须明显更轻，不能只是换一个标题。

## 交互

- 左侧导航高亮当前视图。
- 切换视图时只显示对应的内容区块。
- 页面刷新后保留上次使用的视图。

## 测试

- 页面 HTML 包含导航和视图标记。
- 页面脚本语法通过 `node --check`。
- 全量测试通过。
