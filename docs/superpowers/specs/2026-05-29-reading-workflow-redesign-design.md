# 阅读工作流再设计

## 背景

这次调整接在“首页概览 + 今日情报工作台”之后，重点解决两个问题：

- 首页右侧信息过多，向下滚动时左侧和中部显空。
- 今日情报卡片内容长短不一，四列排布显拥挤，不利于快速扫读。

## 设计目标

- 首页继续保留总体、概览、可视化内容，但让右侧摘要用更均衡的矩阵排列。
- 近期趋势图改成更醒目的主图，历史点不足时用展示填充数据补齐 7 天走势。
- 今日情报列表从四列改成三列，主线数量不变。
- 情报卡片固定高度，只展示足够判断的信息；完整摘要、正文、判断和相关条目放到详情抽屉。
- 今日队列顶部增加摘要，让用户先知道当前筛选下有多少条、多少未读、多少必看、多少稍后看。

## 页面结构

### 首页

- `overview-balance-grid` 将分类占比、来源健康、近期趋势、本周沉淀、观察雷达组织成两列摘要区。
- `overviewSourceTrend` 升级为横跨两列的趋势主卡，避免右侧都是等宽小卡。
- 趋势图使用面积图加折线和节点，强化“近期走势”的可读性。

### 今日情报

- `todayQueueSummary` 放在今日队列标题下方，显示当前队列、未读、必看、稍后看。
- `feed-grid` 默认三列，卡片固定为 330px 高。
- 卡片标题、摘要、判断、标签和元信息都做行数限制，防止某一条撑高整行。
- 详情抽屉使用 `detail-shell`、`detail-hero`、`detail-grid` 拆分标题动作区、主内容区和侧边判断区。
- 抽屉里支持稍后看、归档、收藏、忽略，并在操作后刷新列表与抽屉内容。

## 验证

- 布局测试检查新增结构标记和三列固定卡片规则。
- 浏览器预览检查首页视觉平衡、趋势图、今日队列摘要、三列卡片和详情抽屉。
- 保持已有 API、筛选、标记和打开原文行为不变。
