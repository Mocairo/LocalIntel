# 观察雷达设计

## 目标

把“每天看一批信息”升级为“持续盯住我关心的对象”。第一版只做稳定闭环：配置观察对象、日报更新后匹配当天条目、可选使用大模型判断异动、首页展示缓存结果。

## 范围

- 观察对象写在 `interests.toml` 的 `[[watchlist]]` 中。
- 每个观察对象包含：`id`、`name`、`type`、`keywords`、`description`、`enabled`。
- 支持类型先不做复杂分支，第一版统一按关键词匹配标题、摘要、正文、标签和来源。
- 更新情报后生成 `watch_radar` 缓存表。
- 首页新增“观察雷达”区块，展示最多 6 个观察对象。
- 大模型只在生成期判断，不在页面刷新时调用。

## 数据流

1. `load_watchlist()` 从 `interests.toml` 读取观察对象。
2. `build_watch_radar()` 根据当天 `items` 做本地匹配和评分。
3. 如果 LLM 可用，给模型最多 8 个观察对象候选，让它判断是否异动、原因和建议动作。
4. 结果写入 SQLite。
5. `/api/watch-radar` 只读取 SQLite 缓存。

## 回退策略

- 没有观察对象：接口返回空数组，页面隐藏区块。
- LLM 失败：保留本地规则判断，记录 `llm_jobs`。
- 模型输出不合法：回退本地规则，不影响日报生成。

## 第一版不做

- 不做复杂表单编辑。
- 不做桌面通知。
- 不做跨周趋势图。
- 不把所有条目实时送给模型。

## 验证

- 单元测试覆盖观察清单读取。
- 单元测试覆盖规则匹配和雷达行生成。
- 单元测试覆盖 SQLite 缓存读写。
- 前端脚本语法检查通过。
- 全量 pytest 通过。
