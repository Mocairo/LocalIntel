# 概览页 UI 重新设计说明

## 背景

当前主页已经拆成“概览、今日情报、观察雷达、来源状态”四个视图，但概览页仍然像多个模块直接纵向堆叠，信息密度偏高。用户希望首页更接近仪表盘：只放总体、概览、可视化内容，把详细列表留给其他页面。

## 设计目标

- 首页第一屏要回答：今天有没有情报、运行是否正常、哪些类别/来源/观察对象最值得看。
- 保留现有后端接口和数据结构，只调整前端 HTML、CSS、JS 渲染。
- 视觉风格按工作台处理：安静、清晰、可扫描，不做营销式大图或装饰。
- 遵循 `ui-ux-pro-max` 的关键规则：触控目标不小于 44px、焦点状态可见、移动端不横向滚动、图表不用颜色单独表达含义。

## 页面结构

概览页新增一个独立的 `overview-shell`：

- 顶部总控区 `overviewHero`：显示日报日期、运行状态、下一次自动更新和两个快捷入口。
- KPI 区 `overviewKpis`：承接已有 5 张指标卡，但视觉收敛到总控台内部。
- 运行状态区：继续使用已有 `runtimePanel`，但作为概览的一部分。
- 摘要区：继续使用已有 `llmPanel`，标题改成“今日摘要”语义。
- 高价值信号区：承接已有提醒卡片，概览页最多展示 6 条。
- 三个可视化摘要卡片：
  - `overviewCategoryMix`：分类占比条形图。
  - `overviewSourceHealth`：来源健康状态摘要。
  - `overviewWatchBrief`：观察雷达活跃/静默状态摘要。

## 交互

- 总控区快捷按钮可切到“今日情报”和“观察雷达”视图。
- 可视化卡片只做概览，不替代详细页面。
- 没有数据时展示空状态，不隐藏整个概览。

## 验证

- 增加结构测试，确认概览组件和渲染函数存在。
- 运行现有测试，避免破坏已有接口和页面视图切换。
- 启动本地预览，检查桌面端首屏和移动端宽度不溢出。
