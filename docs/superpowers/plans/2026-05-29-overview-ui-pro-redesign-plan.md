# 概览页 UI 重新设计执行计划

## 成功标准

- 概览页拥有独立的总控台结构，而不是旧模块堆叠。
- 分类、来源、观察雷达都有可视化摘要。
- 详细列表仍留在“今日情报”“观察雷达”“来源状态”页面。
- 测试通过，本地预览能正常打开。

## 步骤

1. 写测试
   - 在 `tests/test_dashboard_layout.py` 中断言新结构和新渲染函数存在。
   - 验证：先运行目标测试，确认失败。

2. 改页面结构
   - 在 `app/web.py` 中用 `overview-shell` 包住概览内容。
   - 新增总控区和三个可视化摘要卡片。
   - 验证：目标测试进入下一类失败或通过。

3. 改样式
   - 补充概览网格、图表条、状态条、快捷按钮、响应式样式。
   - 保持卡片半径不超过 8px，保证触控目标和焦点状态。
   - 验证：页面无明显横向溢出。

4. 改渲染逻辑
   - 基于 `/api/stats` 返回值渲染概览简介、分类占比、来源健康、观察雷达摘要。
   - 快捷按钮复用现有 `setView`。
   - 验证：目标测试和全量测试通过。

5. 预览和提交
   - 重启 `http://127.0.0.1:8769/` 预览。
   - 截图检查桌面和移动端。
   - 提交到当前分支。
